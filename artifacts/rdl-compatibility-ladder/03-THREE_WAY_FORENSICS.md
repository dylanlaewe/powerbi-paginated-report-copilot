## Regression boundary

`assertReportBuilderGroupedHierarchy` accepts only the observed Report Builder-authored grouped shape and rejects both:

- Candidate 02, because it lacks the intermediate `Region1` grouping/header level; and
- Candidate 03, because it has three body rows/leaves and places its group header in a merged body cell.

The regression additionally requires:

- group order `Region → Region1 → Details`;
- row-hierarchy header placement;
- exactly one `KeepWithGroup` marker;
- one Visibility element without an explicit Hidden child;
- the Report Builder `rd:Selected` marker;
- Region sorting on the outer and intermediate groups; and
- SaleDate then Salesperson sorting on Details.

These checks distinguish the independently accepted and rejected structures. They establish a compatibility boundary, not proof that any one token caused the exception.
